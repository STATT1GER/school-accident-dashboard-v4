# 학교안전 사고경로 관제 대시보드 v4 설계·수정 명세

## 1. 개편 목적

v3는 합성 데이터의 분포를 폭넓게 탐색하는 EDA형 대시보드에 가까웠다. v4는 다음 질문에 더 직접적으로 답하도록 개편한다.

> 어떤 시간에, 어떤 장소에서, 어떤 활동을 하다가, 어떤 사고형태로 이어지는가?

따라서 사고경로를 화면의 핵심 정보로 두고 시간·장소·활동 분석은 경로를 해석하기 위한 보조 근거로 사용한다.

이 UI는 특정 날짜의 사고를 예측하는 모델이 아니다. 과거 날짜를 이용해 미래의 동일 날짜를 예측한다는 의미를 사용하지 않으며, 합성 데이터에서 반복되는 연결 구조를 상대 위험신호로 제시한다.

---

## 2. 전역 원칙

### 2.1 사고경로 우선

핵심 분석 단위는 다음 4단계이다.

```text
사고시간 → 사고장소 → 사고당시활동 → 사고형태
```

위험장소 순위는 중요하지만 사고경로보다 하위 정보로 배치한다.

### 2.2 심각도 제거

보상데이터 또는 진단정보가 결합되지 않았으므로 사고심각도를 추정하거나 표시하지 않는다.

제거 대상:

- 사고심각도 도넛 차트
- 주의사고 건수
- 심각도 기반 위험가중치
- 지도상 심각도 색상
- 데이터 미리보기의 심각도 필드

상대 위험도는 합성 신호의 빈도 집중도와 경로 집중도를 이용한 UI 지표로만 표현한다.

### 2.3 합성 데이터 명시

- 실제 사고 원자료 미포함
- 실제 학교·학생·사고ID 미포함
- 특정 미래 날짜 예측 아님
- 화면 시연용 상대 위험도임을 명시

## 3. 메인 `app.py`

### 3.1 히어로 패널

변경사항:

- 검은색 라운드 패널의 제목 글씨 축소
- 패널 높이와 내부 여백 축소
- 좌우 열 비율을 지도 중심으로 변경
- 오른쪽 가상학교 안전지도 확대

권장 열 비율:

```python
st.columns([0.76, 1.24])
```

### 3.2 메시지 변경

기존 데이터 분포 탐색보다 다음 메시지를 강조한다.

- 사고가 발생하기 전의 흐름
- 4단계 사고경로
- 위험신호와 개입 지점
- 예측모델이 아닌 관제 UI

## 4. 오늘의 학교 안전상황

### 4.1 연도 중심 요약 제거

연도 선택 및 월별 추이를 제거한다. 특정 연도 EDA보다 전체 합성 패턴의 반복 위험경로를 현재 운영 관점에서 보여준다.

### 4.2 화면 우선순위

1. 상대 위험도와 핵심 위험시간·장소
2. 가장 중요한 4단계 사고경로
3. 반복 사고경로 TOP 3
4. 학교 일과 상대 위험 흐름
5. 운영 개입 제안
6. 보조 위험장소 순위

### 4.3 위험장소 순위 위계 조정

위험장소 우선순위는 삭제하지 않지만 사고경로보다 아래에 배치하고 보조 정보로 명시한다.

## 5. 학교지도

### 5.1 연도 선택 제거

학교지도는 연도별 EDA가 아니라 현재 운영 시나리오의 공간 위험을 보여주는 화면이므로 연도 필터를 제거한다.

### 5.2 지도 배경 일치

메인 화면의 `assets/school_map.png`를 학교지도 히트맵의 배경으로 그대로 사용한다.

좌표 체계:

```text
지도 X: 0~100
지도 Y: 0~100
```

배경 이미지와 위험 좌표를 동일 스케일로 배치한다.

### 5.3 사고시간 선택 개선

개별 사고시간을 여러 개 선택하는 드롭다운을 제거하고 다음 일과 구간을 가로형 선택 버튼으로 제공한다.

- 전체 일과
- 등교·하교
- 수업시간
- 쉬는시간
- 점심시간
- 방과후

### 5.4 층 선택 가시성 개선

층 선택도 가로형 버튼으로 제공한다.

- 1층
- 2층
- 3층
- 4층
- 전체

선택 상태는 연한 파란 배경과 테두리로만 표시하여 과도하게 강조하지 않는다.

### 5.5 지도 위험도

심각도 기반 색상을 제거하고 다음 기준으로 표현한다.

- 히트맵: 좌표 밀도
- 마커 크기: 위험신호 건수
- 마커 색: 선택 조건 내 상대 빈도 0~100

## 6. 사고분석

### 6.1 탭 순서

```text
사고경로 → 시간 분석 → 장소 분석 → 활동·형태
```

사고분석 화면을 열면 사고경로가 기본 첫 탭으로 보인다.

### 6.2 사고경로 학년군 탭

사고경로 탭 안에 다음 탭을 제공한다.

- 전체
- 저학년
- 고학년

저학년은 1~2학년, 고학년은 3~6학년이다.

### 6.3 Sankey 간단히·자세히

각 학년군 화면에서 두 가지 보기를 제공한다.

#### 간단히 보기

- 단계별 상위 5개 범주
- 나머지는 `기타`로 통합
- 핵심 흐름을 빠르게 읽는 용도

#### 자세히 보기

- 전체 범주 유지
- 작은 연결도 표시
- 세부 경로 탐색 용도

두 보기 모두 다음 4단계를 유지한다.

```text
사고시간 → 사고장소 → 사고당시활동 → 사고형태
```

### 6.4 시간 분석

제거:

- 월별 사고 추이

유지·확대:

- 시간대별 상대 위험 프로파일
- 요일 × 사고시간 히트맵

두 차트 모두 좌우 분할 없이 전체 너비로 세로 배치한다.

### 6.5 장소 분석

기존 장소 막대차트와 층별 분포의 배치는 유지한다.

단, 상세표에서 심각도 관련 열을 제거하고 다음을 제공한다.

- 사고건수
- 비율
- 대표활동
- 대표사고형태

### 6.6 활동·형태

변경 흐름:

1. 사고장소 선택
2. 선택 장소의 사고당시활동 빈도 막대차트
3. 사고당시활동 선택
4. 선택 활동의 사고형태 빈도 막대차트

제거:

- 사고심각도 도넛
- 활동 × 사고형태 히트맵

교사가 교차표를 해석하지 않아도 장소 → 활동 → 형태의 순서로 이해하도록 단순화한다.

## 7. VS Code 및 실행환경

v3의 `.vscode/settings.json`은 생성 전부터 다음 경로를 강제했다.

```text
.venv\Scripts\python.exe
```

이 때문에 `.venv` 생성 전 인터프리터 경고가 발생했다.

v4에서는 `python.defaultInterpreterPath`를 제거한다. 먼저 다음 명령으로 환경을 생성한다.

```powershell
uv sync
```

이후 VS Code에서 `.venv\Scripts\python.exe`를 선택한다.

실행 명령:

```powershell
uv run python -m streamlit run app.py
```
